## Installation

```bash
npx shadcn@latest add @fulldev/theme-toggle
```

## Setup

The Theme Toggle requires the Theme Toggle Provider to be placed in your document's `<head>` to prevent flash of unstyled content (FOUC) and persist theme preferences.

Add the provider to your layout's head section:

```astro
---
import { ThemeToggleProvider } from "@/components/ui/theme-toggle"
---

<html>
  <head>
    <ThemeToggleProvider />
    <!-- other head elements -->
  </head>
  <body>
    <!-- your content -->
  </body>
</html>
```

The theme toggle provider contains the inline bootstrap script from the shadcn Astro dark
mode docs and should be rendered before the page content to avoid a flash of
unstyled theme state.

## Usage

```ts
import { ThemeToggle } from "@/components/ui/theme-toggle"
```

```astro
<ThemeToggle />
```

## Notes

- `ThemeToggle` is a two-state light and dark switch.
- `ThemeToggleProvider` reads the saved preference first and falls back to the
  system color scheme on first load.
- If you need a richer theme picker, build that separately on top of the same
  `.dark` class convention.
