## Installation

### Command

```bash
npx shadcn@latest add @fulldev/combobox
```

### Manual

1. Install the dependency:

```bash
npm install @data-slot/combobox
```

2. Copy the files from `src/components/ui/combobox/` into your project.

3. Update the import paths to match your project setup.

## Composition

### Simple

A single-line input and a flat list.

```text
Combobox
├── ComboboxInput
├── ComboboxTrigger
└── ComboboxContent
    └── ComboboxList
        ├── ComboboxEmpty
        ├── ComboboxItem
        └── ComboboxItem
```

### Popup Input

Use `ComboboxValue` in the trigger and move `ComboboxInput` into the popup when
you want a committed value text target and a separate search input.

```text
Combobox
├── ComboboxTrigger
│   └── ComboboxValue
└── ComboboxContent
    ├── ComboboxInput
    └── ComboboxList
        ├── ComboboxEmpty
        ├── ComboboxItem
        └── ComboboxItem
```

### Groups

Group related items with `ComboboxGroup`, `ComboboxLabel`, and
`ComboboxSeparator`.

```text
Combobox
├── ComboboxInput
└── ComboboxContent
    └── ComboboxList
        ├── ComboboxEmpty
        ├── ComboboxGroup
        │   ├── ComboboxLabel
        │   ├── ComboboxItem
        │   └── ComboboxItem
        ├── ComboboxSeparator
        └── ComboboxGroup
            ├── ComboboxLabel
            └── ComboboxItem
```

### Portal

Use an authored portal and positioner when you want explicit popup placement
markup instead of the generated wrapper from `@data-slot/combobox`.

```text
Combobox
├── ComboboxInput
├── ComboboxTrigger
└── ComboboxPortal
    └── ComboboxPositioner
        └── ComboboxContent
            └── ComboboxList
```

## Selected Value Text

Use `ComboboxValue` when the selected value should render inside the trigger and
the search input should stay inside the popup.

```astro
<Combobox class="w-full max-w-sm" defaultValue="nl">
  <ComboboxTrigger
    class="border-input bg-background dark:bg-input/30 focus-visible:border-ring focus-visible:ring-ring/50 inline-flex h-9 w-full items-center justify-between gap-2 rounded-md border px-3 text-sm shadow-xs transition-[color,box-shadow] outline-none focus-visible:ring-3"
  >
    <ComboboxValue>Select country...</ComboboxValue>
    ▼
  </ComboboxTrigger>
  <ComboboxContent>
    <ComboboxInput placeholder="Search countries..." />
    <ComboboxList>
      <ComboboxEmpty>No countries found.</ComboboxEmpty>
      <ComboboxItem value="nl">Netherlands</ComboboxItem>
      <ComboboxItem value="be">Belgium</ComboboxItem>
      <ComboboxItem value="de">Germany</ComboboxItem>
    </ComboboxList>
  </ComboboxContent>
</Combobox>
```

In popup-input mode:

- `ComboboxValue` stays in sync with the committed value.
- The popup input is cleared every time the popup opens.
- Closing the popup keeps the popup input empty.

## Clear Button

Use `ComboboxClear` to clear the current value while preserving the popup open
state exactly as `@data-slot/combobox` defines it.

```astro
<Combobox class="relative w-full max-w-sm" defaultValue="cet" name="timezone">
  <ComboboxInput class="pr-18" placeholder="Choose a timezone..." />
  <div class="absolute top-1 right-1 flex items-center gap-1">
    <ComboboxClear
      class="text-muted-foreground hover:bg-accent hover:text-accent-foreground focus-visible:ring-ring/50 size-7 rounded-md transition-colors outline-none focus-visible:ring-3"
      aria-label="Clear timezone"
    >
      ✕
    </ComboboxClear>
    <ComboboxTrigger
      class="text-muted-foreground hover:bg-accent hover:text-accent-foreground focus-visible:ring-ring/50 size-7 rounded-md px-0 transition-colors outline-none focus-visible:ring-3"
      aria-label="Toggle timezones"
    >
      ▼
    </ComboboxTrigger>
  </div>
  <ComboboxContent>
    <ComboboxList>
      <ComboboxItem value="cet">Central European Time</ComboboxItem>
      <ComboboxItem value="est">Eastern Standard Time</ComboboxItem>
      <ComboboxItem value="pst">Pacific Standard Time</ComboboxItem>
    </ComboboxList>
  </ComboboxContent>
</Combobox>
```

## Runtime API

The Astro wrappers auto-discover roots with `create()`, so you can still use
the `@data-slot/combobox` DOM events for programmatic control.

```html
<div data-slot="combobox" id="framework-combobox">...</div>

<script>
  const root = document.querySelector("#framework-combobox")

  root?.addEventListener("combobox:change", (event) => {
    console.log(event.detail.value)
  })

  root?.dispatchEvent(
    new CustomEvent("combobox:set", {
      detail: { value: "astro", open: true },
    })
  )
</script>
```

See the [`@data-slot/combobox`](https://www.npmjs.com/package/@data-slot/combobox)
