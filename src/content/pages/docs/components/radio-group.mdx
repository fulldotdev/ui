```astro live
  </div>
  <div class="flex items-center gap-3">

    <Label for="option-two">Option Two</Label>
  </div>
</RadioGroup>
```

## About

`RadioGroup` intentionally stays Data Slot-backed in this library.

- The Data Slot runtime provides compound-part styling hooks, roving keyboard
  focus, and group-level selection behavior.
- It also generates hidden native radios when `name` is provided, so form
  submission and native validation still work.
- This keeps `radio-group` aligned with the library's compound-control pattern,
  while `checkbox` remains a native browser input.

## Installation

### Command

```bash
npx shadcn@latest add @fulldev/radio-group
```

### Manual

1. Install the following dependency:

```bash
npm install @data-slot/radio-group
```

2. Copy and paste the radio group component files into
   `src/components/ui/radio-group/`.

3. Update the import paths to match your project setup.

## Composition

Use the following composition to build a `RadioGroup`:

```text
RadioGroup
├── RadioGroupItem
└── RadioGroupItem
```

## Examples

### Description

Radio group items with a title and supporting copy using the `Field` component.

```astro live
---
import {
  Field,
  FieldContent,
  FieldDescription,
  FieldLabel,
} from "@/components/ui/field"
import { RadioGroup, RadioGroupItem } from "@/components/ui/radio-group"
---

<RadioGroup defaultValue="starter" name="plan" class="w-full max-w-sm">
  <Field orientation="horizontal">
    <RadioGroupItem id="starter" value="starter" />
    <FieldContent>
      <FieldLabel for="starter">Starter</FieldLabel>
      <FieldDescription>Basic features for personal projects.</FieldDescription>
    </FieldContent>
  </Field>
  <Field orientation="horizontal">
    <RadioGroupItem id="pro" value="pro" />
    <FieldContent>
      <FieldLabel for="pro">Pro</FieldLabel>
      <FieldDescription>Advanced tooling for growing teams.</FieldDescription>
    </FieldContent>
  </Field>
</RadioGroup>
```

### Choice Card

Use the wrapping-label pattern from `@data-slot/radio-group` so the entire card

```astro live
        </FieldDescription>
      </FieldContent>
    </Field>
  </FieldLabel>
```

### Fieldset

Use `FieldSet` and `FieldLegend` to group radio items with a shared label and
description.

```astro live
---
import {
  Field,
  FieldDescription,
  FieldLabel,
  FieldLegend,
  FieldSet,
} from "@/components/ui/field"
import { RadioGroup, RadioGroupItem } from "@/components/ui/radio-group"
---

<div class="w-full max-w-sm">
  <FieldSet>
    <FieldLegend>Notifications</FieldLegend>
    <FieldDescription>Choose how you want to be notified.</FieldDescription>
    <RadioGroup defaultValue="email" name="notifications">
      <Field orientation="horizontal">
        <RadioGroupItem value="email" id="notifications-email" />
        <FieldLabel for="notifications-email">Email</FieldLabel>
      </Field>
      <Field orientation="horizontal">
        <RadioGroupItem value="sms" id="notifications-sms" />
        <FieldLabel for="notifications-sms">SMS</FieldLabel>
      </Field>
      <Field orientation="horizontal">
        <RadioGroupItem value="push" id="notifications-push" />
        <FieldLabel for="notifications-push">Push notification</FieldLabel>
      </Field>
    </RadioGroup>
  </FieldSet>
</div>
```

### Invalid

Use `data-invalid="true"` on `Field` for error messaging, and
`aria-invalid="true"` on `RadioGroupItem` when you need assistive technology to
announce the invalid state.

```astro live
---
import {
  Field,
  FieldContent,
  FieldDescription,
  FieldLabel,
} from "@/components/ui/field"
import { RadioGroup, RadioGroupItem } from "@/components/ui/radio-group"
---

<RadioGroup name="support" class="w-full max-w-sm">
  <Field orientation="horizontal" data-invalid="true">
    <RadioGroupItem id="support-email" value="email" aria-invalid="true" />
    <FieldContent>
      <FieldLabel for="support-email">Email</FieldLabel>
      <FieldDescription class="text-destructive">
        Select one contact method before continuing.
      </FieldDescription>
    </FieldContent>
  </Field>
  <Field orientation="horizontal">
    <RadioGroupItem id="support-phone" value="phone" />
    <FieldContent>
      <FieldLabel for="support-phone">Phone</FieldLabel>
      <FieldDescription>We will call during business hours.</FieldDescription>
    </FieldContent>
  </Field>
</RadioGroup>
```

## RTL

The underlying runtime supports directional layout. Apply `dir="rtl"` to a
parent container to render the group in right-to-left mode.

```astro live
---
import { Label } from "@/components/ui/label"
import { RadioGroup, RadioGroupItem } from "@/components/ui/radio-group"
---

<div dir="rtl" class="w-full max-w-sm">
  <RadioGroup defaultValue="email" name="rtl-demo">
    <div class="flex items-center gap-3">
      <RadioGroupItem value="email" id="rtl-email" />
      <Label for="rtl-email">البريد الإلكتروني</Label>
    </div>
    <div class="flex items-center gap-3">
      <RadioGroupItem value="sms" id="rtl-sms" />
      <Label for="rtl-sms">رسالة نصية</Label>
    </div>
  </RadioGroup>
</div>
```
