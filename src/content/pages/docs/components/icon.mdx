This project supports both `@lucide/astro` and `simple-icons-astro`.

- Use `@lucide/astro` for general-purpose UI icons.
- Use `simple-icons-astro` for brand icons like GitHub, Vercel, Discord, and
  other logos.

Both libraries ship Astro components directly, so you can import only the icons
you need and render them like any other Astro component.

```astro live props={{ name: 'icon' }}
---
import Check from "@lucide/astro/icons/check"
import Heart from "@lucide/astro/icons/heart"
import MessageCircle from "@lucide/astro/icons/message-circle"
import Star from "@lucide/astro/icons/star"
import Trash2 from "@lucide/astro/icons/trash-2"
import Github from "simple-icons-astro/Github"
---

<Heart />
<Star />
<Check />
<Trash2 />
<MessageCircle />
<Github fill="currentColor" />
```

## Installation

Install the icon packages you want to use:

```bash
pnpm add @lucide/astro simple-icons-astro
```

## Usage

For UI icons:

```ts
import Heart from "@lucide/astro/icons/heart"
```

```astro
<Heart />
```

For brand icons:

```ts
import Github from "simple-icons-astro/Github"
```

```astro
<Github fill="currentColor" />
```

## Example

Inside buttons and similar controls, use `data-icon` for Lucide icons so the
surrounding component can size and align the icon consistently.

```astro live
---
import ArrowRight from "@lucide/astro/icons/arrow-right"

import { Button } from "@/components/ui/button"
---

<Button>
  Continue
  <ArrowRight data-icon="inline-end" />
</Button>
```

## Notes

- Import only the icons you need.
- Prefer `@lucide/astro` for interface icons and `simple-icons-astro` for brand
  marks.
- Prefer passing icon components directly instead of storing string keys in
  content or config.
