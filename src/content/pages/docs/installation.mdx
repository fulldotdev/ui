Fulldev UI is distributed as a shadcn-compatible `@fulldev` registry for Astro
projects. Components and blocks install as source files in your project.

## Start with an AI agent

Give your AI agent the Markdown version of the installation docs:

```text
Read https://ui.full.dev/index.md, then install Fulldev UI in this Astro project and add @fulldev/button and @fulldev/section.
```

For a specific component or block, give the AI agent that page directly:

```text
Read https://ui.full.dev/components/button.md, then install and use @fulldev/button in this project.
```

Every documentation page has a Markdown version. Add `.md` to the URL to read
the source.

## Create new Astro project

If you don't have a project yet, start from Astro's minimal template. In the
interactive Astro wizard this is the "minimal (empty)" template; in a command it
is named `minimal`.

```bash
pnpm create astro@latest my-project --template minimal --install --no-git --yes
cd my-project
pnpm astro add tailwind -y
pnpm add class-variance-authority clsx tailwind-merge
```

Add the `@/*` import alias and registry config that the installable source
expects:

```json title="tsconfig.json" {3-8} showLineNumbers
{
  "extends": "astro/tsconfigs/strict",
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  },
  "include": [".astro/types.d.ts", "**/*"],
  "exclude": ["dist"]
}
```

```json title="components.json"
{
  "$schema": "https://ui.shadcn.com/schema.json",
  "style": "new-york",
  "rsc": false,
  "tsx": true,
  "tailwind": {
    "config": "",
    "css": "src/styles/global.css",
    "baseColor": "neutral",
    "cssVariables": true,
    "prefix": ""
  },
  "aliases": {
    "components": "@/components",
    "utils": "@/lib/utils",
    "ui": "@/components/ui",
    "lib": "@/lib",
    "hooks": "@/hooks"
  },
  "registries": {
    "@fulldev": "https://ui.full.dev/r/{name}.json"
  }
}
```

Add the shared class helper:

```ts title="src/lib/utils.ts"
import { clsx, type ClassValue } from "clsx"
import { twMerge } from "tailwind-merge"

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs))
}
```

## Add to existing Astro project

Because Fulldev UI uses Shadcn-compatible registry tooling, first make sure the
project has Tailwind CSS v4, the `@/*` import alias, and the `components.json`
file shown above. It also needs `class-variance-authority`, `clsx`, and
`tailwind-merge` installed, plus the `src/lib/utils.ts` helper shown above.

### Check styles

`pnpm astro add tailwind -y` creates `src/styles/global.css` with Tailwind. Keep
that file imported from your app layout or page. Fulldev UI components use the
semantic tokens installed with each registry item.

### Check tsconfig.json

If you did not already add the alias above, add it to `tsconfig.json`:

```json title="tsconfig.json" {4-9} showLineNumbers
{
  "compilerOptions": {
    // ...
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
    // ...
  }
}
```

### Check components.json

If you did not already create `components.json`, add the file shown above.

If you are reading a local development copy of these docs, use that same local
server for the registry while testing. The local dev server normally prints
`http://127.0.0.1:4321/`. In that case, change only the registry URL:

```json title="components.json"
{
  "registries": {
    "@fulldev": "http://127.0.0.1:4321/r/{name}.json"
  }
}
```

Once the registry is present, all install commands in this docs site should work
as written.

### Use a container-aware app shell

Fulldev UI uses Tailwind CSS v4 container query variants such as `@2xl:` and `@max-5xl:`. These are intentional and should not be converted to viewport breakpoints like `2xl:`.

To make those variants work, add `@container` to the root wrapper that contains your page content:

```astro title="src/layouts/Layout.astro" {2,6} showLineNumbers
---
import "@/styles/global.css"
---

<body class="@container">
  <slot />
</body>
```

## Add Components

You can now start adding components to your project.

```bash
npx shadcn@latest add @fulldev/button @fulldev/section -y
```

or multiple resources at once:

```bash
npx shadcn@latest add @fulldev/button @fulldev/card @fulldev/section -y
```

If you are reinstalling over files that already exist, add `--overwrite`.

### Add all components

```bash
npx shadcn@latest add @fulldev/layout @fulldev/components
```

### Usage

The commands above will add components to your project. You can then import them like this:

```astro title="src/pages/index.astro" {2-4,10-31} showLineNumbers
---
import { Button } from "@/components/ui/button"
import { Section, SectionContainer } from "@/components/ui/section"

import "@/styles/global.css"
---

<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width" />
    <title>Astro + Fulldev UI</title>
  </head>
  <body class="@container">
    <Section>
      <SectionContainer class="min-h-screen place-items-center text-center">
        <div class="grid max-w-xl gap-4">
          <h1 class="text-4xl font-semibold tracking-tight">
            Astro + Fulldev UI
          </h1>
          <p class="text-muted-foreground">
            A minimal Astro page using installed Fulldev UI source files.
          </p>
          <div>
            <Button href="/docs/">Get started</Button>
          </div>
        </div>
      </SectionContainer>
    </Section>
  </body>
</html>
```

At that point, you can follow the component pages directly and install only the
pieces you need.

## Validate the install

Run a production build after wiring the first installed components into a page:

```bash
pnpm build
```

## Use with shadcn/ui

Fulldev UI keeps Shadcn-style installation and composition where it fits
Astro. You can use Fulldev UI for content-driven Astro pages and complete page
sections, and still use shadcn/ui when a more interactive part of the project
needs React.
