Fulldev supports two icon workflows:

- Use the Fulldev `Icon` component when an icon name comes from content,
  frontmatter, or project configuration.
- Import `lucide-static` or `simple-icons` components directly when the
  icon is code-owned UI.

The `Icon` component resolves Lucide icons first and falls back to Simple Icons
for brand marks. Direct imports remain useful for fixed interface icons in
buttons, controls, and examples.

```astro live props={{ name: 'icon' }}
---
import { Icon } from "@/components/ui/icon"
---

<Icon name="rocket" />
<Icon name="github" />
```

## Installation

Install the Fulldev icon component when you want content-owned icon names:

```bash
npx shadcn@latest add @fulldev/icon
```

Install direct icon packages only when you need to import icon components in
code:

```bash
pnpm add lucide-static simple-icons
```

## Direct imports

<Alert>
  <AlertTitle>Use path imports for fixed UI icons</AlertTitle>
  <AlertDescription>
    Import fixed Lucide and Simple Icons components from their icon-specific
    paths, such as `lucide-static/icons/check.svg` or
    `simple-icons/icons/github.svg`. Avoid named imports from the package
    entrypoint because they can make the dev server scan much more icon code and
    noticeably slow startup and HMR. The Fulldev `Icon` component uses static
    SVG packages for content-owned dynamic icon names.
  </AlertDescription>
</Alert>

```astro live
---
import CheckIcon from "lucide-static/icons/check.svg"
import HeartIcon from "lucide-static/icons/heart.svg"
import MessageCircleIcon from "lucide-static/icons/message-circle.svg"
import StarIcon from "lucide-static/icons/star.svg"
import Trash2Icon from "lucide-static/icons/trash-2.svg"
import GithubIcon from "simple-icons/icons/github.svg"
---

<HeartIcon />
<StarIcon />
<CheckIcon />
<Trash2Icon />
<MessageCircleIcon />
<GithubIcon fill="currentColor" />
```

## Usage

For content-owned icons, pass plain names to `Icon`:

```astro
<Icon name={feature.icon} />
```

For UI icons that are fixed in code:

```ts
import HeartIcon from "lucide-static/icons/heart.svg"
```

```astro
<HeartIcon />
```

For brand icons:

```ts
import GithubIcon from "simple-icons/icons/github.svg"
```

```astro
<GithubIcon fill="currentColor" />
```

## Example

Inside buttons and similar controls, use `data-icon` for Lucide icons so the
surrounding component can size and align the icon consistently.

```astro live
---
import ArrowRightIcon from "lucide-static/icons/arrow-right.svg"

import { Button } from "@/components/ui/button"
---

<Button>
  Continue
  <ArrowRightIcon data-icon="inline-end" />
</Button>
```

## Notes

- Use plain icon names in content, frontmatter, or project configuration.
- Render content-owned icon names through `Icon`.
- Import icon components directly for fixed code-owned UI icons.
- Prefer `lucide-static` for interface icons and `simple-icons` for brand
  marks when importing directly.
